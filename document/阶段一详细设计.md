## 1. MCP 服务端详细设计

### 1.1 MCPServer 类设计

MCPServer 是整个系统的入口，负责通过 stdio 与 AI 助手通信，同时通过 WebSocket 与 Blender 插件通信。

#### 1.1.1 类接口定义

```python
class MCPServer:
    """MCP 服务端，通过 stdio 与 AI 助手通信，通过 WebSocket 与 Blender 通信。"""

    def __init__(self, config: dict):
        """
        Args:
            config: 配置字典，包含 server/blender/logging/security/limits 等键
        """

    async def start(self) -> None:
        """启动 MCP 服务，初始化 stdio 通信和 WebSocket 客户端连接。"""

    async def stop(self) -> None:
        """优雅停止服务，关闭所有连接和资源。"""

    async def handle_request(self, request: dict) -> dict:
        """
        处理来自 AI 助手的 MCP 请求。

        Args:
            request: JSON-RPC 2.0 格式的请求字典

        Returns:
            JSON-RPC 2.0 格式的响应字典
        """

    async def send_to_blender(self, message: dict) -> dict:
        """
        通过 WebSocket 向 Blender 插件发送消息并等待响应。

        Args:
            message: 操作消息字典

        Returns:
            Blender 插件的响应字典

        Raises:
            ConnectionError: WebSocket 未连接或发送失败
            TimeoutError: 等待响应超时
        """

    def get_server_info(self) -> dict:
        """
        获取服务端信息。

        Returns:
            {
                'name': 'blender-mcp',
                'version': '0.1.0',
                'protocol_version': '2024-11-05',
                'capabilities': {'tools': {'listChanged': True}},
                'connection_state': 'CONNECTED' | 'DISCONNECTED' | ...,
                'blender_info': {...} | None
            }
        """
```

#### 1.1.2 MCP 协议握手流程

MCP 协议基于 JSON-RPC 2.0，初始化序列如下：

```
AI 助手 (stdio)                    MCPServer
     │                                  │
     │─── initialize ──────────────────▶│  {jsonrpc:"2.0", id:1,
     │                                  │   method:"initialize",
     │                                  │   params:{protocolVersion:"2024-11-05",
     │                                  │            capabilities:{},
     │                                  │            clientInfo:{name:"trae-solo",version:"1.0"}}}
     │                                  │
     │◀── initialize result ───────────│  {jsonrpc:"2.0", id:1,
     │                                  │   result:{protocolVersion:"2024-11-05",
     │                                  │            capabilities:{tools:{listChanged:True}},
     │                                  │            serverInfo:{name:"blender-mcp",version:"0.1.0"}}}
     │                                  │
     │─── notifications/initialized ──▶│  {jsonrpc:"2.0",
     │                                  │   method:"notifications/initialized"}
     │                                  │
     │─── tools/list ─────────────────▶│  {jsonrpc:"2.0", id:2,
     │                                  │   method:"tools/list"}
     │                                  │
     │◀── tools/list result ──────────│  {jsonrpc:"2.0", id:2,
     │                                  │   result:{tools:[{name:"ping",...}]}}
```

#### 1.1.3 JSON-RPC 消息格式

**请求消息：**
```json
{
    "jsonrpc": "2.0",
    "id": "req-001",
    "method": "tools/call",
    "params": {
        "name": "ping",
        "arguments": {}
    }
}
```

**成功响应：**
```json
{
    "jsonrpc": "2.0",
    "id": "req-001",
    "result": {
        "content": [
            {
                "type": "text",
                "text": "{\"success\": true, \"latency_ms\": 12.5}"
            }
        ]
    }
}
```

**错误响应：**
```json
{
    "jsonrpc": "2.0",
    "id": "req-001",
    "error": {
        "code": -32602,
        "message": "Invalid params",
        "data": {"detail": "Missing required parameter: object_id"}
    }
}
```

**JSON-RPC 错误码映射：**

| MCP 场景 | JSON-RPC 错误码 | 说明 |
|----------|----------------|------|
| 解析错误 | -32700 | JSON 解析失败 |
| 无效请求 | -32600 | 缺少 jsonrpc/method/id 字段 |
| 方法未找到 | -32601 | tools/call 中的工具名不存在 |
| 无效参数 | -32602 | 参数校验失败 |
| 内部错误 | -32603 | 服务端内部异常 |

#### 1.1.4 stdio 通信层设计

MCP SDK 通过 stdio 与 AI 助手通信，消息以换行符分隔的 JSON 行传输。

```python
class StdioTransport:
    """stdio 通信层，处理消息的读取、解析、分发和响应。"""

    def __init__(self, message_handler: Callable[[dict], Awaitable[dict]]):
        """
        Args:
            message_handler: 异步消息处理回调，接收请求字典，返回响应字典
        """

    async def read_message(self) -> dict | None:
        """
        从 stdin 读取一行 JSON 消息。

        Returns:
            解析后的请求字典，EOF 时返回 None

        Raises:
            json.JSONDecodeError: JSON 格式无效
            ValueError: 消息超过最大大小限制 (10MB)
        """

    async def write_message(self, message: dict) -> None:
        """
        将响应字典序列化为 JSON 并写入 stdout。

        Args:
            message: 响应字典
        """

    async def run(self) -> None:
        """
        主循环：持续从 stdin 读取消息，调用 handler 处理，写回响应。
        遇到 EOF 或 KeyboardInterrupt 时退出。
        """

    async def send_notification(self, method: str, params: dict = None) -> None:
        """
        发送服务端通知（无 id 字段的单向消息）。

        Args:
            method: 通知方法名，如 "notifications/tools/list_changed"
            params: 通知参数
        """
```

**消息读取流程：**

```
stdin 读取一行
    │
    ├─ 检查消息大小 > 10MB? ──是──▶ 丢弃，记录 WARN 日志
    │
    ├─ JSON 解析
    │   ├─ 失败 ──▶ 返回 -32700 Parse error
    │   └─ 成功
    │
    ├─ 验证 JSON-RPC 2.0 字段
    │   ├─ 缺少 jsonrpc/method ──▶ 返回 -32600 Invalid Request
    │   └─ 有效
    │
    ├─ 路由分发
    │   ├─ method == "initialize" ──▶ handle_initialize()
    │   ├─ method == "tools/list" ──▶ handle_tools_list()
    │   ├─ method == "tools/call" ──▶ handle_tools_call()
    │   └─ 其他 ──▶ 返回 -32601 Method not found
    │
    └─ 写回响应到 stdout
```

### 1.2 工具注册机制

#### 1.2.1 ToolRegistry 类设计

```python
from dataclasses import dataclass
from typing import Callable, Any

@dataclass
class ToolDefinition:
    """MCP 工具定义"""
    name: str
    description: str
    input_schema: dict
    handler: Callable[[dict], Awaitable[dict]]

class ToolRegistry:
    """工具注册表，管理所有 MCP 工具的注册、查找和调用。"""

    def __init__(self):
        self._tools: dict[str, ToolDefinition] = {}

    def register(
        self,
        name: str,
        description: str,
        input_schema: dict,
        handler: Callable[[dict], Awaitable[dict]]
    ) -> None:
        """
        注册一个 MCP 工具。

        Args:
            name: 工具名称，如 "ping"
            description: 工具描述
            input_schema: JSON Schema 格式的参数定义
            handler: 异步处理函数，接收参数字典，返回结果字典

        Raises:
            ValueError: 工具名已存在或参数无效
        """

    def unregister(self, name: str) -> bool:
        """
        注销一个工具。

        Returns:
            是否成功注销（工具不存在返回 False）
        """

    def get_tool(self, name: str) -> ToolDefinition | None:
        """
        按名称查找工具。

        Returns:
            工具定义，不存在返回 None
        """

    def list_tools(self) -> list[dict]:
        """
        获取所有已注册工具的列表（MCP tools/list 格式）。

        Returns:
            [{"name": "ping", "description": "...", "inputSchema": {...}}, ...]
        """

    async def call_tool(self, name: str, arguments: dict) -> dict:
        """
        调用指定工具。

        Args:
            name: 工具名称
            arguments: 工具参数

        Returns:
            工具执行结果

        Raises:
            ToolNotFoundError: 工具不存在
            InvalidParameterError: 参数校验失败
        """

    def tool(
        self,
        name: str,
        description: str,
        input_schema: dict
    ) -> Callable:
        """
        装饰器方式注册工具。

        用法:
            @registry.tool("ping", "连接验证", {"type": "object", ...})
            async def handle_ping(args: dict) -> dict:
                return {"success": True}
        """
```

#### 1.2.2 工具注册流程

```
MCPServer.__init__()
    │
    ├─ 创建 ToolRegistry 实例
    │
    ├─ 注册阶段一工具:
    │   ├─ registry.register("ping", ping_handler)
    │   └─ ...
    │
    ├─ (阶段二注册更多工具)
    │
    └─ 启动时通过 tools/list 暴露给 AI 助手
```

#### 1.2.3 工具调用流程

```
AI 助手发送 tools/call 请求
    │
    ├─ MCPServer.handle_request()
    │   ├─ 解析 method = "tools/call"
    │   ├─ 提取 name 和 arguments
    │   │
    │   ├─ ToolRegistry.call_tool(name, arguments)
    │   │   ├─ 查找工具 ──不存在──▶ 返回 ToolNotFoundError
    │   │   ├─ 校验参数 ──失败──▶ 返回 InvalidParameterError
    │   │   ├─ 调用 handler(arguments)
    │   │   │   ├─ handler 内部通过 send_to_blender() 转发
    │   │   │   └─ 等待 Blender 响应
    │   │   └─ 返回结果
    │   │
    │   └─ 格式化为 MCP 响应返回
    │
    └─ 响应通过 stdio 返回 AI 助手
```

```python
_classes = [
    BLENDER_MCP_PT_ConnectionPanel,
    BLENDER_MCP_OT_StartServer,
    BLENDER_MCP_OT_StopServer,

def register():
    """注册插件所有类和属性。"""
    for cls in _classes:
        bpy.utils.register_class(cls)
    bpy.types.Scene.blender_mcp_props = bpy.props.PointerProperty(
        type=BlenderMCPProperties
    )
    logger.info("Blender MCP 插件已注册")

def unregister():
    """注销插件所有类和属性，清理资源。"""
    ws_client = BlenderWSClient.get_instance()
    if ws_client and ws_client.is_connected:
        ws_client.disconnect()

    for cls in reversed(_classes):
        bpy.utils.unregister_class(cls)
    del bpy.types.Scene.blender_mcp_props
    logger.info("Blender MCP 插件已注销")
```

### 2.3 WebSocket 客户端设计

#### 2.3.1 BlenderWSClient 类设计

```python
import asyncio
import json
import threading
import time
import bpy
from typing import Optional

class BlenderWSClient:
    """Blender 端 WebSocket 客户端，与 MCP 服务通信。"""

    _instance: Optional['BlenderWSClient'] = None

    @classmethod
    def get_instance(cls) -> Optional['BlenderWSClient']:
        """获取单例实例。"""

    def __init__(self):
        self._ws = None
        self._host = "127.0.0.1"
        self._port = 8765
        self._connected = False
        self._loop: asyncio.AbstractEventLoop | None = None
        self._thread: threading.Thread | None = None
        self._request_id = 0
        self._pending: dict[int, asyncio.Future] = {}
        self._heartbeat_mgr: HeartbeatManager | None = None
        self._reconnect_config = ReconnectConfig()
        self._connection_state = ConnectionState.DISCONNECTED

    @property
    def is_connected(self) -> bool:
        """是否已连接。"""

    def connect(self, host: str = "127.0.0.1", port: int = 8765) -> None:
        """
        建立 WebSocket 连接。

        在独立线程中运行 asyncio 事件循环，避免阻塞 Blender 主线程。

        Args:
            host: MCP 服务地址
            port: MCP 服务端口

        Raises:
            ConnectionError: 连接失败
        """

    def disconnect(self) -> None:
        """
        断开 WebSocket 连接，停止心跳和事件循环线程。
        """

    def send_request(self, message: dict, timeout: float = 30.0) -> dict:
        """
        同步发送请求并等待响应（Blender 操作在主线程调用）。

        Args:
            message: 请求消息字典，如 {"action": "ping"}
            timeout: 等待超时（秒）

        Returns:
            响应消息字典

        Raises:
            ConnectionError: 未连接
            TimeoutError: 等待响应超时
        """

    async def _async_send_request(self, message: dict, timeout: float) -> dict:
        """异步发送请求的内部实现。"""

    async def _message_handler(self, websocket) -> None:
        """
        WebSocket 消息接收循环。

        接收消息后根据 type 字段分发：
        - "response": 匹配 pending 请求的 Future
        - "event": 触发事件回调
        - "heartbeat": 心跳响应
        """

    async def _heartbeat_loop(self, websocket) -> None:
        """心跳循环，定期发送 Ping 帧。"""

    def _on_connection_lost(self) -> None:
        """连接丢失回调，触发重连状态机。"""

    async def _reconnect(self) -> None:
        """执行重连逻辑，指数退避。"""
```

#### 2.3.2 消息处理流程

```
MCP 服务 → WebSocket → BlenderWSClient
    │
    ├─ 接收 JSON 消息
    │
    ├─ 解析消息类型
    │   ├─ type == "request" ──▶ 执行 bpy 操作，返回响应
    │   │   ├─ action == "create_object" ──▶ bpy.ops.mesh.primitive_*_add()
    │   │   ├─ action == "transform_object" ──▶ obj.location/rotation_euler/scale = ...
    │   │   ├─ action == "ping" ──▶ 返回 {"success": True, "latency_ms": ...}
    │   │   └─ 其他 ──▶ 返回 {"success": False, "error": "UNKNOWN_ACTION"}
    │   │
    │   ├─ type == "event" ──▶ 触发事件回调（阶段三实现）
    │   │
    │   └─ type == "heartbeat" ──▶ 更新心跳时间戳
    │
    └─ 异常处理
        ├─ bpy 异常 ──▶ 返回 {"success": False, "error": "INTERNAL_ERROR", ...}
        └─ JSON 解析异常 ──▶ 记录日志，丢弃消息
```

#### 2.3.3 线程模型

```
Blender 主线程 (UI 线程)
    │
    ├─ 用户操作 → UI 面板按钮 → connect() / disconnect() / send_request()
    │
    ├─ send_request() 通过 asyncio.run_coroutine_threadsafe() 提交到事件循环线程
    │
    └─ 响应通过 bpy.app.timers 或 queue 回调到主线程更新 UI
```

#### 5.4.1 MCP 服务端 ping handler

```python
async def handle_ping(arguments: dict) -> dict:
    """
    ping 工具处理函数。

    流程:
    1. 记录发送时间戳
    2. 通过 WebSocket 向 Blender 插件发送 ping 请求
    3. 等待 Blender 响应
    4. 计算往返延迟
    5. 返回结果

    Args:
        arguments: {"message": "hello"} (可选)

    Returns:
        {
            "success": True,
            "latency_ms": float,
            "blender_version": str,
            "python_version": str,
            "scene_objects": int,
            "mode": str,
            "message": str
        }

    Raises:
        ConnectionError: Blender 未连接
        TimeoutError: 等待响应超时 (5秒)
    """
    start_time = time.monotonic()

    request = {
        "id": generate_message_id(),
        "type": "request",
        "action": "ping",
        "payload": {
            "message": arguments.get("message", ""),
            "timestamp": time.time(),
        },
        "timestamp": iso_now(),
    }

    response = await server.send_to_blender(request, timeout=5.0)

    end_time = time.monotonic()
    latency_ms = (end_time - start_time) * 1000

    if response.get("success"):
        result = response["payload"]
        result["latency_ms"] = round(latency_ms, 2)
        return result
    else:
        return {
            "success": False,
            "latency_ms": round(latency_ms, 2),
            "error": response.get("error", {}).get("message", "Unknown error"),
        }
```

#### 5.4.2 Blender 插件端 ping 处理

```python
def handle_ping_request(payload: dict) -> dict:
    """
    Blender 端 ping 请求处理。

    在 Blender 主线程中执行，收集当前场景状态。

    Args:
        payload: {"message": "hello", "timestamp": ...}

    Returns:
        {
            "success": True,
            "blender_version": "4.2.0",
            "python_version": "3.11.0",
            "scene_objects": 3,
            "mode": "OBJECT",

        }

        "scene_objects": len(bpy.context.scene.objects),

        "message": payload.get("message", ""),

```

```
async call_tool(name, arguments):
    tool = _tools.get(name)
    if tool is None:
        raise ToolNotFoundError(name)

    validate arguments against tool.input_schema
    if validation fails:
        raise InvalidParameterError(details)

    try:
        result = await tool.handler(arguments)
        return result
    except Exception as e:
```

```
    future.result(timeout=10)  # 阻塞等待连接结果

    if connected:
        start_heartbeat()
        start_message_handler()
```
