## 1. save_project 工具详细设计

### 1.1 功能概述

`save_project` 工具负责将当前 Blender 场景保存为 `.blend` 项目文件，支持压缩、备份和路径安全校验。

### 1.2 bpy.ops.wm.save_as_mainfile 调用方式

```python
import bpy
import os

def save_project(
    filepath: str,
    compress: bool = True,
    backup: bool = True,
    relative_remap: bool = True
) -> dict:
    """
    保存 Blender 项目文件。

    Args:
        filepath: 保存路径（必须为绝对路径）
        compress: 是否压缩文件（默认 True）
        backup: 是否创建 .blend.bak 备份（默认 True）
        relative_remap: 是否将外部文件路径转为相对路径（默认 True）

    Returns:
        包含 success、filepath、file_size、backup_created 等字段的字典

    Raises:
        PermissionDeniedError: 路径无写入权限
        PathSecurityError: 路径不在白名单内
        ResourceExhaustedError: 磁盘空间不足
    """
    filepath = os.path.realpath(filepath)

    if not _is_path_in_whitelist(filepath):
        raise PathSecurityError(
            f"路径 '{filepath}' 不在允许的目录白名单内"
        )

    if not os.path.isdir(os.path.dirname(filepath)):
        os.makedirs(os.path.dirname(filepath), exist_ok=True)

    if not _check_disk_space(filepath, estimated_size=_estimate_blend_size()):
        raise ResourceExhaustedError(
            f"磁盘空间不足，无法保存到 '{filepath}'"
        )

    if backup and os.path.exists(filepath):
        backup_path = filepath + ".bak"
        _create_backup(filepath, backup_path)

    bpy.ops.wm.save_as_mainfile(
        filepath=filepath,
        compress=compress,
        relative_remap=relative_remap,
        copy=False,
    )

    file_size = os.path.getsize(filepath)

    return {
        "success": True,
        "filepath": filepath,
        "file_size": file_size,
        "backup_created": backup and os.path.exists(filepath + ".bak"),
        "backup_path": filepath + ".bak" if backup else None,
        "compressed": compress,
    }
```

### 1.3 备份策略（.blend.bak）

| 策略项 | 规则 | 说明 |
|--------|------|------|
| 备份触发条件 | 目标文件已存在时创建备份 | 首次保存无需备份 |
| 备份文件命名 | `{原文件名}.blend.bak` | 与原文件同目录 |
| 备份覆盖 | 每次保存覆盖旧备份 | 仅保留最近一份备份 |
| 备份清理 | 不自动清理 | 用户手动管理备份文件 |
| 备份创建方式 | `shutil.copy2()` 保留元数据 | 原子性复制 |

```python
import shutil

def _create_backup(source_path: str, backup_path: str) -> bool:
    """
    创建备份文件。
    使用临时文件 + rename 实现原子性备份。

    Args:
        source_path: 原始 .blend 文件路径
        backup_path: 备份文件路径

    Returns:
        是否成功创建备份
    """
    temp_backup = backup_path + ".tmp"
    try:
        shutil.copy2(source_path, temp_backup)
        if os.path.exists(backup_path):
            os.remove(backup_path)
        os.rename(temp_backup, backup_path)
        return True
    except OSError:
        if os.path.exists(temp_backup):
            os.remove(temp_backup)
        return False
```

### 1.4 压缩选项

| 选项 | 值 | 效果 | 适用场景 |
|------|------|------|----------|
| `compress=True` | 默认 | Blender 内部使用 gzip 压缩 | 日常保存，减少磁盘占用 |
| `compress=False` | 可选 | 不压缩，文件较大但读写更快 | 大场景频繁保存时提升性能 |

压缩效果参考：

| 场景规模 | 未压缩大小 | 压缩后大小 | 压缩比 | 压缩耗时 |
|---------|-----------|-----------|--------|---------|
| 简单场景（< 100 对象） | ~5 MB | ~2 MB | 40% | < 0.5s |
| 中等场景（100-500 对象） | ~50 MB | ~20 MB | 40% | < 2s |
| 复杂场景（> 500 对象） | ~200 MB | ~80 MB | 40% | < 5s |

### 1.5 路径安全校验

```python
import os
import sys

def _is_path_in_whitelist(filepath: str) -> bool:
    """
    校验文件路径是否在白名单内。
    防止路径遍历攻击（如 ../../../etc/passwd）。

    Args:
        filepath: 待校验的绝对路径（已通过 realpath 规范化）

    Returns:
        路径是否合法
    """
    from config.settings import ALLOWED_PATHS

    resolved = os.path.realpath(filepath)

    if ".." in filepath:
        return False

    for allowed in ALLOWED_PATHS:
        resolved_allowed = os.path.realpath(allowed)
        if sys.platform == 'win32':
            if resolved.lower().startswith(resolved_allowed.lower()):
                return True
        else:
            if resolved.startswith(resolved_allowed):
                return True
    return False

def _check_disk_space(filepath: str, estimated_size: int, safety_factor: float = 1.5) -> bool:
    """
    检查磁盘剩余空间是否足够保存文件。

    Args:
        filepath: 目标路径
        estimated_size: 预估文件大小（字节）
        safety_factor: 安全系数，默认 1.5 倍

    Returns:
        磁盘空间是否充足
    """
    import shutil
    try:
        disk_usage = shutil.disk_usage(os.path.dirname(filepath))
        return disk_usage.free > estimated_size * safety_factor
    except OSError:
        return False

def _estimate_blend_size() -> int:
    """
    估算当前场景保存后的文件大小。
    基于场景中的对象数量和总面数进行粗略估算。

    Returns:
        预估文件大小（字节）
    """
    import bpy
    total_faces = 0
    for obj in bpy.data.objects:
        if obj.type == 'MESH' and obj.data:
            total_faces += len(obj.data.polygons)

    base_size = 1 * 1024 * 1024
    per_face_size = 128
    return base_size + total_faces * per_face_size
```

| 错误场景 | 错误码 | 处理策略 |

## 2. open_project 工具详细设计

### 2.1 功能概述

`open_project` 工具负责加载指定的 `.blend` 项目文件到 Blender 中，处理未保存变更提示、文件锁定检测和场景状态重置。

### 2.2 bpy.ops.wm.open_mainfile 调用方式

```python
import bpy
import os

def open_project(
    filepath: str,
    load_ui: bool = True,
    force: bool = False
) -> dict:
    """
    打开 Blender 项目文件。

    Args:
        filepath: 项目文件路径（必须为绝对路径）
        load_ui: 是否加载 UI 设置（默认 True）
        force: 是否强制打开（忽略未保存变更，默认 False）

    Returns:
        包含 success、filepath、loaded_objects、scene_name 等字段的字典

    Raises:
        ObjectNotFoundError: 文件不存在
        FileLockedError: 文件被其他进程锁定
        UnsavedChangesError: 存在未保存变更且未确认强制打开
        PathSecurityError: 路径不在白名单内
    """
    filepath = os.path.realpath(filepath)

    if not os.path.exists(filepath):
        raise ObjectNotFoundError(f"文件不存在: '{filepath}'")

    if not _is_path_in_whitelist(filepath):
        raise PathSecurityError(f"路径 '{filepath}' 不在允许的目录白名单内")

    if _is_file_locked(filepath):
        lock_info = _get_lock_info(filepath)
        raise FileLockedError(
            f"文件被其他进程锁定: {lock_info['process']} (PID: {lock_info['pid']})"
        )

    if bpy.data.is_dirty and not force:
        raise UnsavedChangesError(
            "当前场景有未保存的变更，请先保存或设置 force=true 强制打开"
        )

    bpy.context.preferences.filepaths.use_scripts_auto_execute = False

    bpy.ops.wm.open_mainfile(
        filepath=filepath,
        load_ui=load_ui,
    )

    loaded_objects = len(bpy.data.objects)
    scene_name = bpy.context.scene.name

    _reset_state_after_load()

    return {
        "success": True,
        "filepath": filepath,
        "loaded_objects": loaded_objects,
        "scene_name": scene_name,
        "load_ui": load_ui,
    }
```

### 2.3 未保存变更提示处理

| 场景 | 处理策略 | 用户操作 |
|------|---------|---------|
| 当前场景有未保存变更 + `force=false` | 返回 `UNSAVED_CHANGES` 错误 | AI 助手提示用户先保存或确认强制打开 |
| 当前场景有未保存变更 + `force=true` | 先自动保存到临时文件，再打开目标项目 | 临时文件路径在响应中返回 |
| 当前场景无未保存变更 | 直接打开 | 无需额外操作 |

```python
def _handle_unsaved_changes(force: bool) -> str | None:
    """
    处理未保存变更。

    Args:
        force: 是否强制打开

    Returns:
        自动保存的临时文件路径（如执行了自动保存），否则 None

    Raises:
        UnsavedChangesError: 存在未保存变更且未强制打开
    """
    if not bpy.data.is_dirty:
        return None

    if not force:
        raise UnsavedChangesError(
            "当前场景有未保存的变更。请先调用 save_project 保存，"
            "或设置 force=true 强制打开（变更将自动保存到临时文件）"
        )

    autosave_dir = os.path.expanduser("~/.blender-mcp/autosave")
    os.makedirs(autosave_dir, exist_ok=True)
    timestamp = time.strftime("%Y%m%d_%H%M%S")
    autosave_path = os.path.join(autosave_dir, f"unsaved_{timestamp}.blend")

    bpy.ops.wm.save_as_mainfile(filepath=autosave_path, compress=True)

    return autosave_path
```

### 2.4 文件锁定检测

```python
import psutil

def _is_file_locked(filepath: str) -> bool:
    """
    检测文件是否被其他进程锁定。

    检测策略：
    1. Windows: 尝试以独占模式打开文件
    2. Linux/macOS: 检查是否有其他 Blender 进程打开了该文件

    Args:
        filepath: 文件路径

    Returns:
        文件是否被锁定
    """
    if sys.platform == 'win32':
        try:
            fd = os.open(filepath, os.O_RDWR | os.O_EXCL)
            os.close(fd)
            return False
        except OSError:
            return True
    else:
        for proc in psutil.process_iter(['pid', 'name', 'open_files']):
            try:
                if proc.info['name'] and 'blender' in proc.info['name'].lower():
                    if proc.info['open_files']:
                        for f in proc.info['open_files']:
                            if os.path.realpath(f.path) == os.path.realpath(filepath):
                                return True
            except (psutil.NoSuchProcess, psutil.AccessDenied):
                continue
        return False

def _get_lock_info(filepath: str) -> dict:
    """
    获取文件锁定信息。

    Returns:
        包含 process 和 pid 字段的字典
    """
    for proc in psutil.process_iter(['pid', 'name', 'open_files']):
        try:
            if proc.info['name'] and 'blender' in proc.info['name'].lower():
                if proc.info['open_files']:
                    for f in proc.info['open_files']:
                        if os.path.realpath(f.path) == os.path.realpath(filepath):
                            return {
                                'process': proc.info['name'],
                                'pid': proc.info['pid'],
                            }
        except (psutil.NoSuchProcess, psutil.AccessDenied):
            continue
    return {'process': 'unknown', 'pid': -1}
```

### 2.5 场景加载后状态重置

```python
def _reset_state_after_load() -> None:
    """
    场景加载后重置 MCP 侧的状态管理器。
    确保状态缓存与新加载的场景一致。
    """
    from core.state import StateManager

    state_manager = StateManager.get_instance()
    state_manager.reset()

    state_manager.version = 0
    state_manager.last_full_sync_time = 0.0
    state_manager.change_log = []

    state_manager.full_sync()

    state_manager.bump('scene_loaded', detail=bpy.context.scene.name)
```

## 3. 状态管理器详细设计

### 3.1 StateManager 类设计

```python
import time
import hashlib
import json
import asyncio
from typing import Optional
from dataclasses import dataclass, field

@dataclass
class ObjectState:
    """单个对象的状态快照"""
    name: str
    type: str
    location: tuple
    rotation_euler: tuple
    scale: tuple
    mesh_hash: str | None
    modifier_count: int
    material_count: int
    visible: bool

@dataclass
class SceneState:
    """场景状态快照"""
    version: int
    timestamp: float
    scene_name: str
    object_count: int
    objects: dict[str, ObjectState]
    scene_hash: str

class StateManager:
    """
    状态管理器。
    维护 Blender 场景状态的缓存，提供增量同步和全量同步能力，
    确保 MCP 侧的缓存与 Blender 实际状态一致。
    """

    _instance: Optional['StateManager'] = None

    def __init__(self):
        self._version: int = 0
        self._last_full_sync_time: float = 0.0
        self._change_log: list[dict] = []
        self._cached_state: Optional[SceneState] = None
        self._object_hashes: dict[str, str] = {}
        self._lock = asyncio.Lock()

        self.FULL_SYNC_INTERVAL = 30.0
        self.MAX_CHANGE_LOG = 100
        self.MAX_SNAPSHOTS = 5
        self._snapshots: list[SceneState] = []

    @classmethod
    def get_instance(cls) -> 'StateManager':
        if cls._instance is None:
            cls._instance = StateManager()
        return cls._instance

    @classmethod
    def reset_instance(cls) -> None:
        cls._instance = None

    @property
    def version(self) -> int:
        return self._version

    @version.setter
    def version(self, value: int) -> None:
        self._version = value

    @property
    def last_full_sync_time(self) -> float:
        return self._last_full_sync_time

    @last_full_sync_time.setter
    def last_full_sync_time(self, value: float) -> None:
        self._last_full_sync_time = value

    @property
    def change_log(self) -> list[dict]:
        return self._change_log

    @change_log.setter
    def change_log(self, value: list[dict]) -> None:
        self._change_log = value

    def reset(self) -> None:
        """重置状态管理器（场景加载后调用）"""
        self._version = 0
        self._last_full_sync_time = 0.0
        self._change_log = []
        self._cached_state = None
        self._object_hashes = {}
        self._snapshots = []
```

### 3.2 场景状态快照机制

```python
def create_snapshot(self, label: str = "") -> str:
    """
    创建当前场景的状态快照。
    快照包含所有对象的元数据和网格哈希。

    Args:
        label: 快照标签

    Returns:
        快照 ID
    """
    import bpy

    snapshot_id = hashlib.md5(
        f"{label}:{bpy.context.scene.name}:{time.time()}".encode()
    ).hexdigest()[:8]

    objects_data = {}
    for obj in bpy.data.objects:
        mesh_hash = None
        if obj.type == 'MESH' and obj.data:
            mesh_hash = self._compute_mesh_hash(obj.data)

        objects_data[obj.name] = ObjectState(
            name=obj.name,
            type=obj.type,
            location=tuple(round(v, 4) for v in obj.location),
            rotation_euler=tuple(round(v, 4) for v in obj.rotation_euler),
            scale=tuple(round(v, 4) for v in obj.scale),
            mesh_hash=mesh_hash,
            modifier_count=len(obj.modifiers),
            material_count=len(obj.material_slots),
            visible=obj.visible_get(),
        )

    snapshot = SceneState(
        version=self._version,
        timestamp=time.time(),
        scene_name=bpy.context.scene.name,
        object_count=len(objects_data),
        objects=objects_data,
        scene_hash=self.compute_scene_hash(),
    )

    self._snapshots.append(snapshot)
    if len(self._snapshots) > self.MAX_SNAPSHOTS:
        self._snapshots.pop(0)

    return snapshot_id

def _compute_mesh_hash(self, mesh) -> str:
    """
    计算网格数据的哈希值。
    基于顶点坐标和面索引生成哈希，用于增量变更检测。

    Args:
        mesh: Blender Mesh 数据对象

    Returns:
        网格哈希值（16 字符十六进制字符串）
    """
    import numpy as np

    if not mesh.vertices:
        return "empty"

    coords = np.array([v.co for v in mesh.vertices], dtype=np.float64)
    coords_rounded = np.round(coords, 4)

    if mesh.polygons:
        face_indices = []
        for poly in mesh.polygons:
            face_indices.extend(poly.vertices)
        face_data = np.array(face_indices, dtype=np.int32)
        combined = np.concatenate([
            coords_rounded.flatten(),
            face_data.flatten().astype(np.float64)
        ])
    else:
        combined = coords_rounded.flatten()

    return hashlib.sha256(combined.tobytes()).hexdigest()[:16]
```

### 3.3 增量同步算法（基于对象哈希的变更检测）

```python
async def incremental_sync(self) -> dict:
    """
    增量同步：仅同步自上次快照以来发生变化的对象。

    算法步骤：
    1. 遍历当前场景中的所有对象
    2. 计算每个对象的哈希值
    3. 与缓存的哈希值比较，识别变更
    4. 仅返回变更的对象信息

    Returns:
        {
            'added': list[str],       # 新增对象名
            'removed': list[str],     # 删除对象名
            'modified': list[str],    # 修改对象名
            'unchanged': list[str],   # 未变化对象名
            'total_objects': int,     # 当前对象总数
        }
    """
    import bpy

    async with self._lock:
        current_objects = {}
        for obj in bpy.data.objects:
            mesh_hash = None
            if obj.type == 'MESH' and obj.data:
                mesh_hash = self._compute_mesh_hash(obj.data)

            current_objects[obj.name] = {
                'type': obj.type,
                'location': tuple(round(v, 4) for v in obj.location),
                'rotation_euler': tuple(round(v, 4) for v in obj.rotation_euler),
                'scale': tuple(round(v, 4) for v in obj.scale),
                'mesh_hash': mesh_hash,
                'modifier_count': len(obj.modifiers),
            }

        cached_names = set(self._object_hashes.keys()) if self._cached_state else set()
        current_names = set(current_objects.keys())

        added = list(current_names - cached_names)
        removed = list(cached_names - current_names)

        modified = []
        unchanged = []
        for name in current_names & cached_names:
            current_hash = current_objects[name].get('mesh_hash')
            cached_hash = self._object_hashes.get(name)
            if current_hash != cached_hash:
                modified.append(name)
            else:
                current_loc = current_objects[name]['location']
                cached_obj = self._cached_state.objects.get(name)
                if cached_obj and (
                    current_loc != cached_obj.location
                    or current_objects[name]['rotation_euler'] != cached_obj.rotation_euler
                    or current_objects[name]['scale'] != cached_obj.scale
                    or current_objects[name]['modifier_count'] != cached_obj.modifier_count
                ):
                    modified.append(name)
                else:
                    unchanged.append(name)

        self._object_hashes = {
            name: data['mesh_hash'] for name, data in current_objects.items()
        }

        if added or removed or modified:
            self.bump('incremental_sync', detail=f"+{len(added)} -{len(removed)} ~{len(modified)}")

        return {
            'added': added,
            'removed': removed,
            'modified': modified,
            'unchanged': unchanged,
            'total_objects': len(current_objects),
        }
```

### 3.4 状态版本号管理

```python
def bump(self, change_type: str, detail: str = "") -> None:
    """
    递增版本号并记录变更。

    Args:
        change_type: 变更类型（如 'object_created', 'scene_loaded'）
        detail: 变更详情
    """
    self._version += 1
    self._change_log.append({
        'version': self._version,
        'type': change_type,
        'detail': detail,
        'timestamp': time.time(),
    })
    if len(self._change_log) > self.MAX_CHANGE_LOG:
        self._change_log.pop(0)

def compute_scene_hash(self) -> str:
    """
    计算当前场景的哈希值，用于快速一致性校验。

    Returns:
        16 字符十六进制哈希字符串
    """
    import bpy

    data = {
        obj.name: {
            'type': obj.type,
            'location': tuple(round(v, 4) for v in obj.location),
            'rotation': tuple(round(v, 4) for v in obj.rotation_euler),
            'scale': tuple(round(v, 4) for v in obj.scale),
        }
        for obj in bpy.data.objects
    }
    serialized = json.dumps(data, sort_keys=True)
    return hashlib.sha256(serialized.encode()).hexdigest()[:16]

def get_changes_since(self, since_version: int) -> list[dict]:
    """
    获取自某个版本以来的所有变更。

    Args:
        since_version: 起始版本号

    Returns:
        变更记录列表
    """
    return [
        c for c in self._change_log
        if c['version'] > since_version
    ]

def is_stale(self, client_version: int) -> bool:
    """
    判断客户端版本是否过期。

    Args:
        client_version: 客户端持有的版本号

    Returns:
        客户端缓存是否需要更新
    """
    return client_version < self._version

def full_sync(self) -> SceneState:
    """
    执行全量同步，获取完整的场景状态。

    Returns:
        完整的场景状态快照
    """
    import bpy

    objects_data = {}
    for obj in bpy.data.objects:
        mesh_hash = None
        if obj.type == 'MESH' and obj.data:
            mesh_hash = self._compute_mesh_hash(obj.data)

        objects_data[obj.name] = ObjectState(
            name=obj.name,
            type=obj.type,
            location=tuple(round(v, 4) for v in obj.location),
            rotation_euler=tuple(round(v, 4) for v in obj.rotation_euler),
            scale=tuple(round(v, 4) for v in obj.scale),
            mesh_hash=mesh_hash,
            modifier_count=len(obj.modifiers),
            material_count=len(obj.material_slots),
            visible=obj.visible_get(),
        )

    self._cached_state = SceneState(
        version=self._version,
        timestamp=time.time(),
        scene_name=bpy.context.scene.name,
        object_count=len(objects_data),
        objects=objects_data,
        scene_hash=self.compute_scene_hash(),
    )

    self._object_hashes = {
        name: obj_state.mesh_hash
        for name, obj_state in objects_data.items()
    }
    self._last_full_sync_time = time.time()

    return self._cached_state
```

### 4.4 事件节流和防抖策略

```python
import time
from collections import defaultdict

class EventThrottle:
    """
    事件节流器。
    防止高频事件（如连续拖拽导致的 depsgraph 更新）淹没通知系统。

    策略：
    1. 节流（Throttle）：同一类型事件在窗口期内只发送一次
    2. 防抖（Debounce）：连续事件停止后才发送最终状态
    3. 合并（Coalesce）：窗口期内的多个同类事件合并为一条通知
    """

    def __init__(
        self,
        throttle_interval: float = 0.1,
        debounce_interval: float = 0.3,
        max_batch_size: int = 50
    ):
        self.throttle_interval = throttle_interval
        self.debounce_interval = debounce_interval
        self.max_batch_size = max_batch_size

        self._last_emit_time: dict[str, float] = defaultdict(float)
        self._pending_events: dict[str, list[SceneChangeEvent]] = defaultdict(list)
        self._debounce_timers: dict[str, float] = {}

    def should_emit(self, event_type: EventType) -> bool:
        """
        判断事件是否应该立即发送。

        Args:
            event_type: 事件类型

        Returns:
            是否应该发送
        """
        now = time.monotonic()
        last = self._last_emit_time.get(event_type.value, 0)

        if now - last >= self.throttle_interval:
            self._last_emit_time[event_type.value] = now
            return True

        return False

    def add_pending(self, event: SceneChangeEvent) -> None:
        """
        添加待发送事件到缓冲区。

        Args:
            event: 场景变更事件
        """
        key = event.event_type.value
        self._pending_events[key].append(event)

        if len(self._pending_events[key]) > self.max_batch_size:
            self._pending_events[key] = self._pending_events[key][-self.max_batch_size:]

    def flush_pending(self, event_type: EventType | None = None) -> list[SceneChangeEvent]:
        """
        刷新待发送事件缓冲区。

        Args:
            event_type: 指定事件类型，None 则刷新全部

        Returns:
            待发送事件列表
        """
        if event_type:
            key = event_type.value
            events = self._pending_events.pop(key, [])
        else:
            events = []
            for key in list(self._pending_events.keys()):
                events.extend(self._pending_events.pop(key, []))

        return events
```

事件节流参数配置：

| 事件类型 | 节流间隔 | 防抖间隔 | 说明 |
|---------|---------|---------|------|
| `OBJECT_MODIFIED` | 100ms | 300ms | 连续变换操作合并 |
| `OBJECT_ADDED` | 0ms | 0ms | 立即发送 |
| `OBJECT_DELETED` | 0ms | 0ms | 立即发送 |
| `MODIFIER_CHANGED` | 200ms | 500ms | 修改器参数调整合并 |
| `SCENE_CHANGED` | 500ms | 1000ms | 场景级变更合并 |
| `UNDO_PERFORMED` | 0ms | 0ms | 立即发送（触发全量同步） |

## 5. 性能优化设计

### 5.1 大场景状态同步优化

| 优化策略 | 实现方式 | 预期效果 |
|---------|---------|---------|
| 增量同步 | 仅同步哈希值变化的对象 | 大场景下同步数据量减少 90%+ |
| 分层哈希 | 对象级哈希 → 场景级哈希 | 快速判断是否需要同步 |
| 延迟计算 | 首次访问时才计算网格哈希 | 避免不必要的哈希计算 |
| 采样哈希 | 大网格仅采样部分顶点计算哈希 | 50 万面以上模型哈希计算 < 100ms |
| 批量通知 | 合并短时间内的多个事件 | 减少通知频率 80%+ |

```python
def _compute_mesh_hash_sampled(self, mesh, sample_rate: float = 0.1) -> str:
    """
    对大网格进行采样哈希计算。
    当网格顶点数超过阈值时，仅采样部分顶点计算哈希。

    Args:
        mesh: Blender Mesh 数据对象
        sample_rate: 采样率（0.0-1.0），默认 10%

    Returns:
        网格哈希值
    """
    import numpy as np

    MESH_HASH_FULL_THRESHOLD = 100000

    vertex_count = len(mesh.vertices)
    if vertex_count < MESH_HASH_FULL_THRESHOLD:
        return self._compute_mesh_hash(mesh)

    coords = np.array([v.co for v in mesh.vertices], dtype=np.float64)
    sample_size = max(int(vertex_count * sample_rate), 1000)
    indices = np.linspace(0, vertex_count - 1, sample_size, dtype=int)
    sampled = coords[indices]
    sampled_rounded = np.round(sampled, 4)

    return hashlib.sha256(sampled_rounded.tobytes()).hexdigest()[:16]
```

### 5.2 增量更新 vs 全量更新策略

| 触发条件 | 同步策略 | 数据量 | 延迟 |
|---------|---------|--------|------|
| WebSocket 连接建立 | 全量同步 | 完整场景状态 | < 2s |
| WebSocket 重连 | 全量同步 | 完整场景状态 | < 3s |
| 场景切换 | 全量同步 | 完整场景状态 | < 2s |
| 用户执行 undo/redo | 全量同步 | 完整场景状态 | < 1s |
| 增量同步检测到不一致 | 全量同步 | 完整场景状态 | < 2s |
| 单个对象创建/删除/修改 | 增量同步 | 仅变更对象 | < 100ms |
| 修改器参数变更 | 增量同步 | 仅变更对象 | < 100ms |
| 定期同步（30s） | 增量同步 | 仅变更对象 | < 100ms |

```python
async def smart_sync(self, client_version: int | None = None) -> dict:
    """
    智能同步：根据条件自动选择增量或全量同步。

    Args:
        client_version: 客户端持有的版本号，None 表示需要全量同步

    Returns:
        同步结果
    """
    now = time.time()

    if client_version is None:
        return await self._do_full_sync()

    if now - self._last_full_sync_time > self.FULL_SYNC_INTERVAL:
        return await self._do_full_sync()

    if not self.is_stale(client_version):
        return {'sync_type': 'none', 'version': self._version}

    changes = self.get_changes_since(client_version)
    has_complex_changes = any(
        c['type'] in ('undo_performed', 'scene_loaded', 'scene_changed')
        for c in changes
    )

    if has_complex_changes:
        return await self._do_full_sync()

    return await self._do_incremental_sync()
```

### 5.3 缓存策略

| 缓存类型 | 缓存内容 | 失效条件 | 最大容量 |
|---------|---------|---------|---------|
| 对象哈希缓存 | 对象名 → 网格哈希 | 对象修改、删除 | 无限制 |
| 场景状态缓存 | 完整 SceneState | 任何场景变更 | 1 份（当前状态） |
| 快照缓存 | SceneState 列表 | FIFO 淘汰 | 5 份 |
| 变更日志 | 版本变更记录 | FIFO 淘汰 | 100 条 |
| 查询结果缓存 | get_scene_info 结果 | 场景变更时失效 | 1 份（TTL 1s） |

```python
class QueryResultCache:
    """
    查询结果缓存。
    缓存 get_scene_info 等频繁调用的查询结果，
    在场景未变更时直接返回缓存。
    """

    def __init__(self, ttl: float = 1.0):
        self._cache: dict[str, tuple[float, dict]] = {}
        self._ttl = ttl
        self._last_version: int = 0

    def get(self, key: str, current_version: int) -> dict | None:
        """
        获取缓存结果。

        Args:
            key: 缓存键
            current_version: 当前状态版本号

        Returns:
            缓存结果，如缓存无效返回 None
        """
        if current_version != self._last_version:
            self._cache.clear()
            self._last_version = current_version
            return None

        if key in self._cache:
            timestamp, data = self._cache[key]
            if time.time() - timestamp < self._ttl:
                return data
            del self._cache[key]

        return None

    def set(self, key: str, value: dict, current_version: int) -> None:
        """
        设置缓存结果。

        Args:
            key: 缓存键
            value: 缓存值
            current_version: 当前状态版本号
        """
        self._last_version = current_version
        self._cache[key] = (time.time(), value)
```
